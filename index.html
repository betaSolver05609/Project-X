About Project X

Project X is a high-performance vector database that allows developers to ingest and search embeddings efficiently. Unlike heavy database engines, it is lightweight and optimized for AI workflows.

It is designed for AI developers, researchers, and engineers who need to ingest, store, and search embeddings efficiently.

Written entirely in modern C++, Project X has a low memory footprint and blazing-fast performance, making it ideal for rapid prototyping, experimentation, and deployment in environments where speed and efficiency are critical.

Getting Started

‚öôÔ∏è System Requirements

  • Operating System: Linux (Ubuntu/Debian recommended) or Windows with WSL2 (Ubuntu/Debian)
  • Dependencies: g++ (C++17 or higher), make, libjsoncpp

The installer will automatically check and install these dependencies if missing.

Installation Steps

# 1. Download the release zip (choose the URL from the Downloads section)
curl -L -o Project-X.zip https://github.com/betaSolver05609/Project-X/releases/download/1.0.1/X.zip

# 2. Extract the package
unzip Project-X.zip

# 3. Move into project directory
cd Project-X/X

# 4. Make the installer executable
chmod +x install.sh

# 5. Run the installer
./install.sh

The installer will automatically check for dependencies (g++, make, libjsoncpp), install them if missing, and then build the database engine.

Start

projectx

This command will start the database CLI.

Note: If you see command not found after installation, add the following line to your ~/.bashrc or ~/.zshrc: 

export PATH="$HOME/.local/bin:$PATH"

Then reload your shell:

source ~/.bashrc

Documentation

At a high level, these are the commands available in the Project X CLI:

Create Insert Ingest Show List Search Drop Alter Exit

1. Create

The CREATE command initializes a Keyspace, which acts as a high-level memory block to store embeddings. A metric must be specified for similarity calculations. If no metric is provided, the default metric is COSINE.

Syntax:

CREATE KEYSPACE <KEYSPACE_NAME>
CREATE KEYSPACE <KEYSPACE_NAME> WITH METRIC <METRIC_NAME>

Supported Metrics: COSINE, EUCLIDEAN, MANHATTAN, CHEBYSHEV, JACCARD, MAHALANOBIS, HAMMING, JSD, MINKOWSKI, DOT

Examples:

>> CREATE KEYSPACE TEST_KEYSPACE
Keyspace 'TEST_KEYSPACE' created with metric 'COSINE'.

>> CREATE KEYSPACE TEST_KEYSPACE1 WITH METRIC EUCLIDEAN
Keyspace 'TEST_KEYSPACE1' created with metric 'EUCLIDEAN'.

2. Insert

You can manually add vectors into a keyspace using the INSERT command.

Syntax:

INSERT <KEYSPACE_NAME> <VECTOR ELEMENTS SEPARATED BY ,>

Example:

>> INSERT TEST_KEYSPACE 1.5,-2.56,3.89
SUCCESS : Inserted record into keyspace 'TEST_KEYSPACE'

>> SHOW RECORDS TEST_KEYSPACE
Record 0: [ 1.5 -2.56 3.89 ]

3. Ingest

Use INGEST to bulk load embeddings from a JSON or CSV file. The database automatically detects the file type and parses it.

JSON Format:

[
    [0.4626082479, -0.0667535439, -0.5117347240],
    ...
]

CSV Format:

0.2570554912,-0.7182904481,0.2366195321
0.0544933564,-0.4744476974,0.5054354071
0.4291793406,-0.0872246176,0.6704185605

Example:

>> INGEST DATA INTO KEYSPACE TEST_KEYSPACE FILEPATH="./tmp/word_embeddings1.csv"
[==================================================] 100% (1000000/1000000)
Ingested 1000000 records into keyspace 'TEST_KEYSPACE'.

4. Show

The SHOW command can be used to view records in a keyspace or inspect its metadata (such as the metric being used).

Syntax:

SHOW RECORDS <KEYSPACE_NAME>
SHOW METRIC FOR <KEYSPACE_NAME>

Example:

>> SHOW METRIC FOR TEST_INGEST
Keyspace 'TEST_INGEST' is using metric 'COSINE'.

>> SHOW RECORDS TEST_INGEST
Record 0: [ 0.257055 -0.71829 0.23662 ]
Record 1: [ 0.0544934 -0.474448 0.505435 ]
Record 2: [ 0.429179 -0.0872246 0.670419 ]
Record 3: [ -0.669465 -0.349562 0.592874 ]

5. List

The LIST command displays all available keyspaces in your database.

Syntax:

LIST KEYSPACES

Example:

>> LIST KEYSPACES
- VECTOR
- TEST_INGEST
- TEST_KEYSPACE

Note: All keyspaces are persisted across sessions until dropped.

6. Search

The SEARCH command queries the database and finds the closest matching records based on the metric defined in the keyspace metadata.

Syntax:

SEARCH <KEYSPACE_NAME> <VECTOR ELEMENTS> TOP <N>

Example:

>> SEARCH TEST_INGEST 1.0,-2.5,3.7 TOP 3
Record Index: 1   | Similarity: 0.977595
Record Index: 115 | Similarity: 0.96859
Record Index: 36  | Similarity: 0.96734

7. Drop

The DROP command deletes a keyspace along with its records and metadata. This action is irreversible in the current release.

Syntax:

DROP KEYSPACE <KEYSPACE_NAME>

Example:

>> DROP KEYSPACE TEST_KEYSPACE
Keyspace 'TEST_KEYSPACE' dropped and file deleted.

8. Alter

The ALTER command modifies the metadata for a keyspace, such as changing the similarity metric used during search queries.

Syntax:

ALTER KEYSPACE <KEYSPACE_NAME> SET METRIC <METRIC_NAME>

Example:

>> SHOW METRIC FOR TEST_INGEST
Keyspace 'TEST_INGEST' is using metric 'COSINE'.

>> SEARCH TEST_INGEST 1.0,-2.5,3.7 TOP 3
Record Index: 1   | Similarity: 0.977595
Record Index: 115 | Similarity: 0.96859
Record Index: 36  | Similarity: 0.96734

>> ALTER KEYSPACE TEST_INGEST SET METRIC EUCLIDEAN
Keyspace 'TEST_INGEST' metric changed to 'EUCLIDEAN'.

>> SEARCH TEST_INGEST 1.0,-2.5,3.7 TOP 3
Record Index: 107 | Similarity: 0.225739
Record Index: 147 | Similarity: 0.222739
Record Index: 91  | Similarity: 0.218535

9. Exit

The EXIT command ends the session and gracefully shuts down the database.

Syntax:

EXIT

Example:

>> EXIT
Exiting. Database saved.
